Onboarding

Setup turns each machine into one known role set.

iMac

Install runtime and app, choose Brain Mac, then issue per-device tokens.

Laptops

Choose Laptop, paste the generated Mac config, then install intercom and worker services.

Room Pi

Run the generated release-pinned Pi installer and verify audio, camera, and systemd with jarvis-pi doctor.

Evidence

Use Collect Evidence in Setup or run jarvis bringup with an output folder, then summarize the folder as the fleet acceptance gate.

Fresh fleet runbook

Bring devices up in the order the brain expects.

1

Install the iMac first, choose Brain Mac, set the private-network host, install services, and keep Setup open for issuing tokens.

2

Issue one Mac config command per laptop from the iMac, paste it on that laptop, choose Laptop, check brain and worker, then install services.

3

Issue the Pi installer from the iMac and run the generated command on the Pi. It must include JARVIS_REF=v0.1.17.

4

Collect evidence on every device, summarize at least four files, then prove one Mac runtime update, one app update, and one Pi update.

Public-ready

Secrets stay local. Devices ask to join.

Per-device tokens are bound to device ids and issued from the brain Mac.

Browser profiles, OAuth caches, user files, and worker jobs are ignored.

Repos stay public-ready only when CI and public-readiness checks pass.
